SQL INJECT

SQL Injection vulnerability is an occurrence whereby an attacker can submit code through web forms that returns db information that doesn't belong to them. Very naughty.

The attacker submits a command to the database that tricks the application into executing unintededed commands. These may include creating, reading, updating, altering or deleting items from your db.

Examples include:

SELECT * FROM Users WHERE UserId = 105 or 1=1

or

SELECT UserId, Name, Password FROM Users WHERE UserId = 105 or 1=1

This passes into the db as true because 'OR 1=1' is always true, yikes!

Newb Programmers should establish best practices right out of the gate to close vulnerabilities. This step alone takes a huge bite out of crime. After setting good security habits it is also important for web developers to continually update their practices based on the latest known vulnerability and their counter measures.
